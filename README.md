# Conversations

Бэкенд-сервис для управления конференциями и бронированиями.

## Требования

- Docker и Docker Compose
- Go 1.21+
- Make

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone git@github.com:MusledPower/conversations.git
cd conversations
```

### 2. Запуск проекта

```bash
# Поднять все сервисы (PostgreSQL, приложение)
make up

# Применить миграции базы данных
make migrate

# Заполнить базу тестовыми данными (опционально)
make seed
```

### 3. Проверка работы

Сервис будет доступен по адресу, указанному в конфигурации `./config/local.yml`

## Доступные команды

### Основные команды

| Команда | Описание |
|---------|----------|
| `make up` | Запустить все Docker-контейнеры |
| `make migrate` | Применить миграции базы данных |
| `make seed` | Заполнить базу тестовыми данными |
| `make help` | Показать список доступных команд |

### Команды для тестирования

| Команда | Описание |
|---------|----------|
| `make test-db-up` | Запустить тестовую БД (порт 5433) |
| `make test-db-down` | Остановить и удалить тестовую БД |
| `make test-migrate-up` | Запустить тестовую БД и применить миграции |
| `make test-migrate-down` | Откатить все миграции в тестовой БД |
| `make test-e2e` | Запустить E2E тесты |
| `make test-clean` | Очистить тестовую БД и временные файлы |

## Конфигурация

Основная конфигурация находится в `./config/local.yml`

### Параметры базы данных

Переменные окружения (настраиваются в `Makefile`):

- `PG_USER` - пользователь PostgreSQL (по умолчанию: `postgres`)
- `PG_PASS` - пароль (по умолчанию: `12345678`)
- `PG_HOST` - хост БД (по умолчанию: `postgres_service`)
- `PG_PORT` - порт БД (по умолчанию: `5432`)
- `PG_NAME` - имя базы данных (по умолчанию: `conv`)


## Разработка

### Запуск в режиме разработки

```bash
# 1. Поднять инфраструктуру
make up

# 2. Применить миграции
make migrate

# 3. Заполнить тестовыми данными
make seed

# 4. Запустить приложение (вручную или через IDE)
go run ./cmd/app/main.go
```

### Тестирование

```bash
# Запустить E2E тесты (автоматически поднимет тестовую БД)
make test-e2e

# Очистить после тестов
make test-clean
```

### Работа с миграциями

Миграции находятся в папке `./migrations` и применяются автоматически командой `make migrate`.

Для тестовой БД:
```bash
# Применить миграции в тестовую БД
make test-migrate-up

# Откатить миграции в тестовой БД
make test-migrate-down
```

## Остановка проекта

```bash
# Остановить все контейнеры
docker compose down

# Остановить и удалить volumes (очистит базу данных)
docker compose down -v
```
